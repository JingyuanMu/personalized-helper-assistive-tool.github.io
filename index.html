SPADE: Smart Personal Assistant for Daily Essentials

Kevin Tieu, Yiyang Yan, Jingyuan Mu
Texas A&M University
Code

Abstract

Smart Personal Assistant for Daily Essentials (SPADE) aims to address the privacy and efficiency challenges of virtual assistants by developing a local Large Language Model (LLM) solution. Unlike traditional virtual assistants like Alexa and Google Assistant, which often compromise user privacy and demand significant computational resources, our approach leverages a local LLM to process user prompts and manage conversations securely on-device. The system interacts with a local database containing contacts and documents, enabling the AI to perform tasks such as sending emails, creating calendar events, summarizing PDFs, and searching the internet via an API. By minimizing reliance on external servers and optimizing for low computational overhead, this project ensures enhanced user privacy and efficient task execution, making virtual assistants more accessible and secure for everyday use.

Motivation

Virtual assistants, such as Google Assistant, Apple Siri, and Amazon Alexa, have become popular in the last decade [1, 2, 3]. However, these products collect user information that adversaries can target to steal and sell sensitive and personal data [4]. With the advancement of artificial intelligence (AI) in recent years, especially in large language models (LLMs), they have shown to possess greater capabilities while demanding less computational resources [5]. Combining the benefits of a smart assistant with the power and efficiency of LLMs, we would like to explore the possibility of developing a personalized AI assistant to help users in every life activities while maintaining the balance of privacy, efficiency, and computational feasibility.

System Overview


Overall, the SPADE system is designed to be modular, local and efficient:
  • - Modular: Each functionality is independent and can be easily maintained and updated.
  • - Local: Utilizing Ollama, the LLM is run locally, ensuring the confidentiality of personal information.
  • - Efficient: By leveraging local LLMs and RAG, SPADE minimizes the programming effort and simplifies the computational complexity.

The system is divided into three main components:
  • - LLM: The LLM is the core of the system, responsible for processing user prompts and managing conversations.
  • - Database: The database contains user information, contacts and documents, which the LLM can access through RAG to perform tasks.
  • - Functionality: The functionality includes all tasks that SPADE can perform. They are modular and independent.

The system is connected together through LangGraph and LangChain framework, alllowing the robust and efficient communication between the components. The user interacts with the system through a simple interface, where they can input their requests. Those requests are processed by the LLM, which decide which functionality to use based on the user input. The current LLM is Llama 3.2 3B model, which is a small but capable LLM that can be run on a small GPU. The LLM then retrieves the necessary information from the database or executes the task directly through the provided functionalities. The current framework supports multiple functionalities: sending emails (via Gmail API), extracting content from documents, searching the web via an API (Tavily API), and scheduling events (Google Calendar API). New functionalities can be easily integrated in the future due to the system's modularity. In the event that the user input is not clear or additional inputs are required, the LLM will ask clarifying questions to ensure that the task is performed correctly.

Limitation


SPADE is a proof-of-concept system that demonstrates the feasibility of using local LLMs for personal assistant tasks. However, it has several limitations that need to be addressed in future work:
  • - Limited Functionality: The current implementation only supports a few functionalities, such as sending emails and scheduling events. Future work should focus on expanding the range of supported tasks and improving the overall user experience.
  • - Performance: The performance of the system may vary depending on the LLM's capabilities. Bigger models will perform better, but will also require more hardware resources (GPU, RAM).
  • - User Interface: The current user interface is basic and could be improved to provide a more intuitive and user-friendly experience.
